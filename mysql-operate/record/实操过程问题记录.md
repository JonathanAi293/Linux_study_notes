**Demo01:对表、数据操作(DDL,DML)**


> 问题:向表插入含中文数据时连续报错(ERROR 1366：中文编码字符串非法)
> 原因：建表未指定`utf8mb4`字符集，表默认字符集为`latin1`，不兼容中文 UTF-8 编码；  
> 解决：建表时直接指定字符集。
> ```
> CREATE TABLE user_table ( 
> id INT(11), 
> name VARCHAR(50), 
> age INT(11),
> gender VARCHAR(1) 
> ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE utf8mb4_unicode_ci;

> 问题：查询拥有员工的部门 ID、部门名称时出现重复结果
> ![输入图片说明](/imgs/2026-07-22/X5IeHsaOoMRoZpbB.png)
> 解决：在选择字段前增加distinct以去除重复结果。

> 问题：在查询要求4时（查询所有年龄大于 40 岁的员工，及其归属的部门名称；如果员工没有分配部门，也需要展示出来。），将年龄小于40的员工也查询出来了，且他们的部门被错误输出成null。
> 原因：将age>=40放入on条件判断，由于两表连接方式为左连接，没有匹配上条件的员工都会显示，且其部门信息为null。
> ![输入图片说明](/imgs/2026-07-22/vSY2yxb6uAVSPV4u.png)
> 解决：将年龄判断放入where。
