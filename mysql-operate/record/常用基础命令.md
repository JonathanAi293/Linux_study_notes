# **Mysql常用基础命令**

**1. 库操作**

查询所有数据库

```
SHOW DATABASES;
```

查询当前数据库
```
SELECT DATABASE();
```

创建数据库
```
CREATE DATABASE [IF NOT EXISTS] 数据库名 [DEFAUT CHARSET字符集] [COLLATE 排序规则]
```

删除
```
DROP DATABASE [IF EXISTS]数据库名 
```

使用
```
USE 数据库名；
```

#
**2. 表操作**

查询当前数据库所有表
```
SHOW TABLES
```

创建表
```
create table 表名(
字段1 字段1类型 [约束] comment 字段1备注,
字段2 字段2类型 [约束] comment 字段2备注,
字段n 字段n类型 [约束] comment 字段n备注，
constraint 外键名称 foreign key(外键字段名) references 主表(主表列名)
)comment 表备注;
```

>示例：
> ```
> create table tb_user( 
> id int comment  primary key auto_increment comment '编号', 
> name varchar(50) comment '姓名',
> age int comment '年龄',
> gender varchar(1) comment '性别'
>)comment '用户表';
约束类型
约束|描述|关键字
--|--|--|
非空约束 | 限制该字段的数据不能为 null|NOT NULL
唯一约束 | 保证该字段的所有数据都是唯一、不重复的 | UNIQUE
主键约束 | 主键是一行数据的唯一标识，要求非空且唯一 | PRIMARY KEY
默认约束 | 保存数据时，如果未指定该字段的值，则采用默认值 | DEFAULT
检查约束 (8.0.16 版本之后)| 保证字段值满足某一个条件 | CHECK
外键约束 | 用来让两张表的数据之间建立连接，保证数据的一致性和完整性 | FOREIGN KEY



查询表结构 
```
DESC 表名；
```

查询指定表的建表语句
```
SHOW CREATE TABLE 表名
```

添加字段
```
alter table 表名 add 字段 类型(长度) comment 注释;
```

修改表中字段类型
```
alter table 表名 modify 字段 新类型(长度) comment 注释;
```

修改表中字段名称和类型
```
alter table 表名 change 旧字段 新字段 新类型(长度) comment 注释;
```

删除字段
```
alter table 表名 drop 字段名;
```

修改表名
```
alter table 表名 rename to 新表名;
```

删除表
```
drop table [if exists] 表名;
```

删除并重新创建该表
```
truncate table 表名;
```

**3. 数据操作**

插入数据
```
insert into 表名(字段1,字段2,字段n) values(值1,值2,值n) ;
```

对所有字段插入数据
```
insert into 表名 values(值1,值2,值n) ;
```

批量插入数据：
```
insert into 表名 values(值1,值2,值n),(值1,值2,值n),(值1,值2,值n);
```

修改数据
```
update 表名 set 字段1=值1,字段n=值n [where 条件];
```

> 如果没有where条件，则表示修改表中所有数据
> 示例：
> ```
> update user_table set age=26 where id=1;

删除数据
```
delete form 表名 [where 条件];
```

**4.查询（DQL）**

查询字段
```
select 字段1,字段2,字段n from 表名 where 条件;
```
```
--以别名显示字段
--distinct去除重复数据;
select [distinct] 字段 '别名' from 表名;
```
```
--查询所有字段
select * from 表名;
```
聚合函数
```
select 聚合函数(字段) from 表名;
```

> 常见聚合函数：count（统计数量）,max,min,sum（求和）,avg（平均值）

分组查询
```
select 字段 from 表名 [where 条件] group by 分组字段名 [having 分组后过滤条件]；
```
```
--示例
--查询年龄小于 45 的员工，并根据工作地址分组，获取员工数量大于等于 3 的工作地址
select workaddress,count(*) from emp where age < 45 group by workaddress having count(*)>=3;
```
排序查询
```
select 字段 from 表名 order by 字段1 排序方式,字段2 排序方式;
```
```
--示例
select * from emp order by age asc , entrydate desc;
```

> asc:升序（默认）；desc:降序
> 字段1相等时，按字段2顺序排序

分页查询
```
select 字段 from 表名 limit 起始索引,查询记录数;
```

> 起始索引=（查询页数-1）*每页记录数

DQL编写顺序

> SELECT 字段列表 
> FROM 表名 
> WHERE 行过滤条件 
> GROUP BY 分组字段 
> HAVING 分组后过滤条件
>ORDER BY 排序字段 
>LIMIT 分页限制

多表查询
```
--内连接
select 字段 from 表1 inner join 表2 on 条件;
```
```
--左外连接
select 字段 from 表1 left outer join 表2 on 条件;
```
```
--右外连接
select 字段 from 表1 right outer join 表2 on 条件;
```
联合查询
```
select 字段 from 表1
union [all]
select 字段 from 表2;
```

> union对结果去重，union all不对结果去重。

**6. 事务**

查看/设置事务提交方式
```
select @@autocommit;
set @@autocommit=0
```
> 0为手动提交，1为自动提交。

开启事务
```
start transaction;
```
```
begin;
```

> 两种方式，二选一。

提交事务
```
commit;
```
回滚事务
```
rollback;
```
事务特性（ACID）
| 特性 |描述  |
|--|--|
原子性（Atomicity）| 事务是不可分割的最小操作单元，要么全部成功，要么全部失败。
一致性（Consistency）| 事务完成时，必须使所有的数据都保持一致状态。
隔离性（Isolation）| 数据库系统提供的隔离机制，保证事务在不受外部并发操作影响的独立环境下运行。
持久性（Durability）| 事务一旦提交或回滚，它对数据库中的数据的改变就是永久的。

并发事务问题
|问题|描述|实例
|--|--|--
脏读 | 一个事务读到另外一个事务还没有提交的数据。|事务A修改数据，未提交。事务B读到已修改数据。
不可重复读 | 一个事务先后读取同一条记录，但两次读取的数据不同，称之为不可重复读。|事务A修改数据，已提交。事务B在提交前后分别读取该数据。
幻读 | 一个事务按照条件查询数据时，没有对应的数据行，但是在插入数据时，又发现这行数据已经存在，好像出现了 "幻影"。|事务A插入一个新数据，未提交。此时事务B插入数据和查询均报错。

事务隔离级别
隔离级别 | 脏读 | 不可重复读 | 幻读
--|--|--|--
Read uncommitted|√|√|√
Read committed|×|√|√
Repeatable Read (默认)|×|×|√
Serializable|×|×|×

> 该表中，√ 表示有可能会发生，× 表示不会发生。

查看事务隔离级别
```
select @@transaction_isolation;
```
设置事务隔离级别
```
set [session | global] transaction isolation level 隔离级别;
```

> session(会话级别)：只对当前客户端窗口有效；
> global：针对所有客户端窗口有效。

explain

> 置于查询语句前，用于查询语句性能。

字段 | 核心含义 | 优化判断标准
--|--|--
id | 查询序号，代表 SQL 执行顺序；多表关联 / 子查询会生成多个 id|id 越大越先执行；id 相同从上到下执行
select_type | 查询类型，区分普通查询、子查询、联合查询 | SIMPLE 最优；DERIVED/UNCACHEABLE SUBQUERY 性能差
table | 当前操作的表名，<derivedN>代表临时表，<unionN>代表联合结果集 | 尽量少出现临时表、union 衍生表
partitions | 匹配的分区表分区，无分区则为 NULL|NULL 正常；大量分区匹配需优化分区条件
**type** | 访问类型（核心优化字段）| 性能优先级：system>const>eq_ref>ref>range>index>ALL，杜绝 ALL 全表扫描
**key** | 实际使用到的索引名称 | NULL 代表没走索引，需加索引
**key_len** | 使用索引的字节长度，判断索引覆盖完整度 | 长度越短越好；联合索引可判断命中前缀
ref | 和索引匹配的对比条件 | const 常量、字段关联、func 函数（函数会失效索引）
rows | 预估扫描行数（预估需要读取多少行数据）| 数值越小性能越好
filtered | 条件过滤后剩余数据百分比 | 越高越好，过低说明 WHERE 过滤效率差
Extra | 额外执行信息，包含关键性能隐患 | 出现 Using filesort/Using temporary 需重点优化

type | 说明
--|--
system | 表仅 1 行数据，最优
const | 主键 / 唯一索引等值匹配
eq_ref | 关联查询主键精准匹配
ref | 普通索引等值匹配
range | 索引范围查询（> < between in）
index | 全索引扫描，比全表扫描略好
ALL | 全表扫描，最差

Extra 值 | 问题 | 优化方案
--|--|--
Using filesort | 文件排序，未利用索引排序 | ORDER BY 字段建立联合索引
Using temporary | 创建临时表存储中间结果 | 优化 GROUP BY，避免无序字段分组
Using index | 覆盖索引，无需回表（优秀）| 保留当前联合索引
Using where | 回表后过滤数据，正常现象 | 可优化索引减少扫描行数
索引使用规则

> **1. 最左前缀法则：** 如果索引了多列（联合索引），要遵守最左前缀法则。最左前缀法则指的是查询从索引的最左列开始，并且不跳过索引中的列。 如果跳跃某一列，索引将部分失效(后面的字段索引失效)。
> **2. 范围查询：** 联合索引中，出现范围查询(>,<)，范围查询右侧的列索引失效。解决方法：在实际允许的情况下，改用>=,<=。
> **3. 对索引列进行运算：** 会使索引失效。
> **4. 查询字符串不加引号：** 会使索引失效。
> **5. 模糊查询：** 头部模糊（例如'%xx'）会使索引失效，尾部模糊（例如'xx%'）不影响。
> **6.or连接的条件：** 如果条件中有任一字段无索引，则索引失效。
